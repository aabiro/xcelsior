## Overview

Instances are the core unit of work on Xcelsior. Submitters create instances specifying their compute requirements, and the scheduler matches them to available GPU hosts.

Examples below use `XCELSIOR_BEARER_TOKEN`, which can be an API key or an OAuth access token.

## Submitting an Instance

```bash
curl -X POST https://xcelsior.ca/instance \
  -H "Authorization: Bearer $XCELSIOR_BEARER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "llama-70b-inference",
    "vram_needed_gb": 48,
    "priority": 8,
    "tier": "sovereign"
  }'
```

### Instance Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Instance identifier |
| `vram_needed_gb` | float | Required GPU VRAM in GB |
| `priority` | int | 1-10 (higher = more urgent) |
| `tier` | string | Isolation tier: `community`, `secure`, `regulated`, `sovereign` |

## Instance Lifecycle

```
queued → assigned → running → completed
                  ↘ failed
         ↘ cancelled
```

| Status | Description |
|--------|-------------|
| `queued` | Waiting for host allocation |
| `assigned` | Matched to a host, starting |
| `running` | Executing on GPU host |
| `completed` | Finished successfully |
| `failed` | Execution error |
| `cancelled` | Cancelled by submitter |

## Scheduling Algorithm

The scheduler uses a 3-step filtering process:

1. **VRAM Check** — Host must have enough free VRAM
2. **Admission Gate** — Only `admitted = true` hosts considered
3. **Isolation Tier** — Sovereign/regulated jobs prefer gVisor/Kata runtime

Trigger scheduling manually:

```bash
POST /queue/process
```

## Spot Instances

Submit interruptible instances at a bid price:

```bash
curl -X POST https://xcelsior.ca/spot/instance \
  -H "Authorization: Bearer $XCELSIOR_BEARER_TOKEN" \
  -d '{
    "name": "batch-embeddings",
    "vram_needed_gb": 8,
    "max_bid": 0.50
  }'
```

Spot instances can be preempted if a higher-priority instance arrives. Run preemption cycles:

```bash
POST /spot/preemption-cycle
```

## Listing Instances

```bash
GET /instances               # All instances
GET /instance/{job_id}       # Single instance details
GET /api/slurm/instances     # Slurm-compatible instance listing
```

## Instance Artifacts

After completion, access instance outputs:

```bash
GET /api/artifacts/{job_id}  # List artifacts
POST /api/artifacts/download # Get presigned download URL
```
