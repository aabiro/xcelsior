## Overview

Xcelsior handles all billing in **Canadian dollars (CAD)** with full 13-province GST/HST/QST support, wallet-based credits, and Stripe Connect payouts to providers.

## Wallet Management

### Check Balance

```bash
GET /api/billing/wallet/{customer_id}
```

### Deposit Credits

```bash
POST /api/billing/wallet/{customer_id}/deposit
Content-Type: application/json

{
  "amount": 100.00,
  "currency": "CAD"
}
```

## Invoicing

Generate a detailed invoice with provincial tax breakdown:

```bash
GET /api/billing/invoice/{customer_id}
```

## CAF Export

Export billing data in CRA Charities and Finance format:

```bash
GET /api/billing/export/caf/{customer_id}
```

Returns a CSV file suitable for Canadian tax filing.

## Refunds

Process refunds for failed jobs or disputes:

```bash
POST /api/billing/refund
Content-Type: application/json

{
  "job_id": "job-123",
  "reason": "host_failure",
  "amount": 5.00
}
```

Host-side failures result in **full automatic refunds**. Submitter-initiated cancellations may receive partial refunds based on compute consumed.

## Spot Instance Metering

Spot (interruptible) instances bill at the **locked spot rate** (`spot_rate_cad`) set when the scheduler assigns a host, not the host's on-demand `cost_per_hour` and not a customer bid.

- **Launch:** Wallet preflight estimates one hour at the live unified spot quote for the requested GPU model.
- **While running:** Usage ticks debit the renter at `spot_rate_cad` × elapsed seconds.
- **On preemption:** Metering stops at `preempted_at`; partial hours are billed pro-rata. The job is requeued with `pricing_mode=spot` preserved; the next assignment may lock a new spot rate.
- **Provider payout:** Marketplace allocations use `gpu_allocations.price_cents_per_hour` for `allocation_type=spot`. Earnings API exposes `spot_earned_cad` vs `on_demand_earned_cad`.

```bash
# Launch spot, rate shown in launch UI / GET /spot-prices
curl -X POST https://xcelsior.ca/instance \
  -H "Authorization: Bearer $XCELSIOR_BEARER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "spot-job", "vram_needed_gb": 8, "pricing_mode": "spot"}'
```

## Provider Payouts

Providers receive payouts via Stripe Connect:

1. **Onboard** via `POST /api/providers/onboard` (KYC/KYB)
2. **Earn** from completed jobs (minus platform fee)
3. **Payout** automatically to connected bank account

### Stripe Connect Flow

```
Submitter pays → Platform holds in escrow → Job completes →
Provider gets (price - platform_fee) → Platform keeps fee
```
