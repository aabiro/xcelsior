## Overview

Xcelsior uses two authentication methods:

1. **Bearer Token** — For API and CLI access
2. **Session Cookie** — For the web dashboard (HttpOnly cookie)

## Bearer Token Authentication

All non-public API endpoints require a Bearer token in the `Authorization` header:

```bash
Authorization: Bearer <token>
```

Accepted bearer token types:

- User API keys
- OAuth access tokens from `xcelsior login` or the browser/device flow
- OAuth `client_credentials` tokens for machine-to-machine callers

### API Key Example

For user-scoped bearer auth, set the `XCELSIOR_API_TOKEN` environment variable in your shell or `.env` file:

```bash
XCELSIOR_API_TOKEN=your-secret-token-here
```

### OAuth Machine Client Example

For worker or service automation, use a confidential OAuth client:

```bash
XCELSIOR_OAUTH_CLIENT_ID=your-client-id
XCELSIOR_OAUTH_CLIENT_SECRET=your-client-secret
```

Exchange it for an access token:

```bash
curl -X POST https://xcelsior.ca/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=$XCELSIOR_OAUTH_CLIENT_ID" \
  -d "client_secret=$XCELSIOR_OAUTH_CLIENT_SECRET" \
  -d "scope=api"
```

### Example API Request

Examples below use `XCELSIOR_BEARER_TOKEN` as a shell variable for whichever bearer token you are using.

```bash
curl -X GET https://xcelsior.ca/api/hosts \
  -H "Authorization: Bearer $XCELSIOR_BEARER_TOKEN"
```

## Public Endpoints

These endpoints do **not** require authentication:

| Endpoint | Description |
|----------|-------------|
| `GET /` | Landing page |
| `GET /docs` | Swagger UI |
| `GET /redoc` | ReDoc |
| `GET /openapi.json` | OpenAPI specification |
| `GET /healthz` | Health check |
| `GET /readyz` | Readiness probe |
| `GET /metrics` | Prometheus metrics |
| `GET /api/stream` | SSE event stream |
| `GET /api/transparency/report` | Transparency report |
| `GET /llms.txt` | LLM-optimized documentation |

## Dashboard Authentication

The web dashboard at `https://xcelsior.ca/dashboard` uses HTTP-only session cookies:

1. Navigate to `https://xcelsior.ca/login`
2. Sign in with email/password or OAuth (GitHub, Google, Hugging Face)
3. A secure `xcelsior_session` cookie is set automatically
4. All subsequent dashboard API calls include the cookie

### OAuth Providers

| Provider | Scopes |
|----------|--------|
| GitHub | `read:user`, `user:email` |
| Google | `openid`, `email`, `profile` |
| Hugging Face | `openid`, `profile` |

## SDK Authentication

When using the TypeScript SDK:

```typescript
import { XcelsiorApiClient, XcelsiorApiEnvironment } from "@xcelsior-gpu/sdk";

const client = new XcelsiorApiClient({
  environment: XcelsiorApiEnvironment.Production,
  headers: {
    Authorization: `Bearer ${process.env.XCELSIOR_BEARER_TOKEN}`,
  },
});
```
