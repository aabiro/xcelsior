## Base URL

| Environment | URL |
|-------------|-----|
| Production | `https://xcelsior.ca` |
| Development | `http://localhost:9500` |

## Authentication

Include a bearer token in the `Authorization` header:

```bash
Authorization: Bearer <token>
```

Bearer auth can use an API key, an OAuth user access token, or an OAuth machine token. See [Authentication](/guides/authentication) for details.

## API Groups

The published API reference is organized into these client-facing groups:

| Group | Description | Key Endpoints |
|-------|-------------|---------------|
| **Hosts** | GPU host registration and management | `PUT /host`, `GET /hosts` |
| **Instances** | Instance submission and lifecycle | `POST /instance`, `GET /instances` |
| **Billing** | Wallets, invoices, refunds | `GET /api/billing/wallet/{id}` |
| **Marketplace** | Rig listings and browsing | `GET /marketplace` |
| **Reputation** | Trust scores and verification | `GET /api/reputation/{id}` |
| **Teams** | Team membership and shared access | `GET /api/teams/me` |
| **Artifacts** | Upload/download presigned URLs | `POST /api/artifacts/upload` |
| **Compliance** | Tax rates and province matrix | `GET /api/compliance/provinces` |
| **Privacy** | PIPEDA consent management | `GET /api/v2/privacy/consents` |
| **OAuth & Sessions** | Authorization Code, Device Flow, and session auth | `GET /oauth/authorize`, `POST /oauth/token` |

Internal worker, admin, maintenance, and callback endpoints are intentionally omitted from the public Fern spec and SDK.

## Response Format

All API responses use JSON:

```json
{
  "status": "ok",
  "data": { ... }
}
```

### Error Responses

```json
{
  "detail": "Error description"
}
```

Standard HTTP status codes:
- `200` — Success
- `201` — Created
- `400` — Bad request
- `401` — Unauthorized
- `403` — Forbidden
- `404` — Not found
- `429` — Rate limited
- `500` — Internal server error

## Rate Limiting

- **30 requests/second** per IP
- **20 concurrent connections** per IP
- **Burst**: 50 requests
- Health endpoints exempt (`/healthz`, `/readyz`)

Rate-limited responses return `429` with a `Retry-After` header.

## Pagination

List endpoints support pagination:

```bash
GET /instances?offset=0&limit=50
```

## SSE Streaming

Real-time events via Server-Sent Events:

```bash
GET /api/stream
```

Events include job status changes, host updates, telemetry, and billing events.
