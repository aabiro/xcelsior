## Base URL

| Environment | URL |
|-------------|-----|
| Production | `https://xcelsior.ca` |
| Development | `http://localhost:9500` |

## Authentication

Include your API token in the `Authorization` header:

```bash
Authorization: Bearer <XCELSIOR_API_TOKEN>
```

See [Authentication](/guides/authentication) for details on public endpoints and OAuth.

## API Groups

The API is organized into these endpoint groups:

| Group | Description | Key Endpoints |
|-------|-------------|---------------|
| **Hosts** | GPU host registration and management | `PUT /host`, `GET /hosts` |
| **Jobs** | Job submission and lifecycle | `POST /job`, `GET /jobs` |
| **Billing** | Wallets, invoices, refunds | `GET /api/billing/wallet/{id}` |
| **Marketplace** | Rig listings and browsing | `GET /marketplace` |
| **Spot Pricing** | Dynamic pricing and preemption | `GET /spot-prices` |
| **Reputation** | Trust scores and verification | `GET /api/reputation/{id}` |
| **Verification** | Hardware attestation | `POST /api/reputation/verify` |
| **SLA** | Uptime tracking and enforcement | `GET /api/sla/status/{id}` |
| **Providers** | Stripe Connect onboarding | `POST /api/providers/onboard` |
| **Artifacts** | Upload/download presigned URLs | `POST /api/artifacts/upload` |
| **Jurisdiction** | Data residency and sovereignty | `GET /api/jurisdiction/trace` |
| **Compliance** | Tax rates and province matrix | `GET /api/compliance/provinces` |
| **Privacy** | PIPEDA consent management | `GET /api/privacy/consents` |
| **Transparency** | Legal requests and canary | `GET /api/transparency/report` |
| **Telemetry** | Real-time GPU metrics | `GET /api/telemetry/{host_id}` |
| **Agent** | Worker agent communication | `POST /agent/versions` |
| **Autoscale** | Auto-scaling pool management | `POST /api/autoscale/pool` |
| **Events** | Event sourcing and state machine | `GET /api/events` |
| **Infrastructure** | Health checks and metrics | `GET /healthz` |

## Response Format

All API responses use JSON:

```json
{
  "status": "ok",
  "data": { ... }
}
```

### Error Responses

```json
{
  "detail": "Error description"
}
```

Standard HTTP status codes:
- `200` ŌĆö Success
- `201` ŌĆö Created
- `400` ŌĆö Bad request
- `401` ŌĆö Unauthorized
- `403` ŌĆö Forbidden
- `404` ŌĆö Not found
- `429` ŌĆö Rate limited
- `500` ŌĆö Internal server error

## Rate Limiting

- **30 requests/second** per IP
- **20 concurrent connections** per IP
- **Burst**: 50 requests
- Health endpoints exempt (`/healthz`, `/readyz`)

Rate-limited responses return `429` with a `Retry-After` header.

## Pagination

List endpoints support pagination:

```bash
GET /jobs?offset=0&limit=50
```

## SSE Streaming

Real-time events via Server-Sent Events:

```bash
GET /api/stream
```

Events include job status changes, host updates, telemetry, and billing events.
